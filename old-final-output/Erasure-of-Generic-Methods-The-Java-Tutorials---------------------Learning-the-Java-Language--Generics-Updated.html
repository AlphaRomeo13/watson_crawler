Erasure of Generic Methods

The Java compiler also erases type parameters in generic method arguments. Consider the following generic method:

WHATSON? 0370a50a-4b32-4caf-86e3-044dcfe6d601

// Counts the number of occurrences of elem in anArray.
//
public static <T> int count(T[] anArray, T elem) {
    int cnt = 0;
    for (T e : anArray)
        if (e.equals(elem))
            ++cnt;
        return cnt;
}

Because T is unbounded, the Java compiler replaces it with Object:

WHATSON? f121d05a-eeb5-49cf-ae14-820a99913249

public static int count(Object[] anArray, Object elem) {
    int cnt = 0;
    for (Object e : anArray)
        if (e.equals(elem))
            ++cnt;
        return cnt;
}

Suppose the following classes are defined:

WHATSON? 6d9597e1-85ea-486a-899c-093db2cf7b80

class Shape { /* ... */ }
class Circle extends Shape { /* ... */ }
class Rectangle extends Shape { /* ... */ }

You can write a generic method to draw different shapes:

WHATSON? 4e19c061-7928-4c6f-abef-5c4358e43a4c

public static <T extends Shape> void draw(T shape) { /* ... */ }

The Java compiler replaces T with Shape:

WHATSON? 959285b0-1f00-4284-bfcf-1c38494fbf73

public static void draw(Shape shape) { /* ... */ }
