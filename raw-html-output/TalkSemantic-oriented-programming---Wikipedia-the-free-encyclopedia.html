Semantic-oriented programming sounds as a tautology in general, given the fact that all programming languages have a semantics (languages do have a semantics by definition). Chide Groenouwe (talk) 23:32, 3 June 2010 (UTC)

The idea behind semantic oriented programming is that, unlike many programming languages, the focus is on semantics not syntax, and that those semantics are malleable things. The "semantic meaning" of something in programming languages is interesting, because semantics in programming languages don't follow your definition. Semantics in programming languages are the actual actions taken by the language for a specific concept. In a loose sense, yes, it is the meaning of the code. In a more formal sense it is a possible thing you can describe. For example pointer semantics are the rules that govern how pointers in a given language work. And the semantic meaning of them can change from language to language, even though the semantics for pointers remain the same (In that they have the same semantic construct, but the meaning of that construct could differ). Semantic is a domain specific word to describe the semantics of programming languages, unfortunely that means it is hard to describe the semantics of semantics, hence semantic meaning. (The domain being programming language design). 199.48.195.202 (talk) 00:24, 5 March 2012 (UTC)
